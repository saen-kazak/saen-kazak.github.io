Visibility

It is important for users to be able to tell exactly what a user interface does simply by looking at it.

Feedback

Feedback should be provided on the status of an action after the action has occurred., such as informing the user that a login was successful or failed.

Affordance

The Affordance principle refers to how easy it is for a user to understand how something is used based on its appearance. For example, a user settings page would have high affordance if the user is aware of its capabilities and what can be achieved through its use.

Mapping

Mapping refers to how well the controls of a design resemble their effect. For example, an arrow telling the user to swipe down would be a good example if it moves the page down.

Constraints

Essentially validation. Inputs should be constrained to stop incorrect values from being entered.

Consistency

The elements of a design should be similar in appearance to minimise confusion. For example, if a webpage uses a specific font style for normal text, this font style should be consistent across the whole web page.
